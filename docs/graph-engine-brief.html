Fief Graph Engine

Mathematical Foundations & Feature Replacement Analysis

Standard engineering solutions to graph problems — subscription cascades, ECS cache invalidation, inference deadlock detection, temporal indexing — are stateful patches on top of a math problem. The math itself, when chosen correctly, is stateless, deterministic, provably convergent, and replayable to arbitrary time T from a seed. This brief evaluates eight feature class replacements on that basis.

stateless evaluation provable termination seedable replay uncertainty as value epistemic-first design
01 Relational Truth Storage → Fuzzy Graph Theory

Replaces binary edge existence checks and ad-hoc weight fields with formal membership values in [0,1] and t-norm composition. Edge weights gain precise semantics: product norm encodes compounding erosion of trust, minimum norm encodes chain-strength limits, Łukasiewicz encodes aggressive decay.

strengths

  • Weight semantics are formally specified, not implicit conventions
  • "Somewhat allied" is a valid state, not a design hack requiring a threshold decision
  • T-norm choice makes world theory explicit and swappable per verb type
  • Composition rules are stateless — no history needed to evaluate a chain

weaknesses

  • T-norm selection is non-obvious; wrong choice means wrong propagation feel
  • Mixed t-norms across verb types require careful documentation
  • Requires content authors to think in [0,1] rather than boolean

opportunities

  • Relational truth on a spectrum becomes a design primitive across all systems
  • Players never "cross a threshold into hostile" — they drift, which is more realistic
  • Partial alliance, tentative trust, and residual fear are all expressible without special cases

threats

  • Designers will demand binary answers; fuzziness resists "is this working" checks
  • QA cannot assert "ally: true" — requires threshold conventions anyway at presentation layer
  • Cognitive overhead for content authors without prior exposure to fuzzy logic
02 Inference & Transitivity → Convergent Policy Functions

Replaces depth caps, explicit cycle detectors, and fixed iteration limits with a requirement that every transitive rule supplies a monotonically decreasing weight function bounded below by zero. Termination is guaranteed by construction. A cycle simply decays to epsilon and stops deriving new edges.

strengths

  • No cycle detection code — cycles terminate mathematically
  • Convergence is provable analytically at rule registration time
  • Termination is a property of the rule, not the runtime
  • Works correctly regardless of graph topology

weaknesses

  • A misconfigured policy function produces wrong propagation silently
  • Analytical convergence verification is non-trivial for exotic functions
  • Designers cannot easily reason about how far an inference "travels"

opportunities

  • Weight decay IS the game mechanic — influence fades with social distance as math, not design
  • Different decay curves per verb type produce meaningfully different political textures
  • Policy functions double as tuning knobs with clear behavioral interpretation

threats

  • Designers may want hard cutoffs ("stop at two hops") that fight the convergent model
  • Very slow convergence on dense graphs may require epsilon threshold tuning
  • Requires policy function library and discipline to not use raw lambdas
03 ECS State Projection → KG Embedding Evaluation

Replaces dirty flags, dependency manifests, version counters, and re-derivation scheduling with entities and relations embedded in a continuous vector space. Relational state is a distance or angle, computed on demand from current graph state. There is no cached value to invalidate — there is only the current geometry.

strengths

  • No invalidation cascade — the embedding is current by construction
  • Conflicting graph state degrades gracefully rather than throwing a conflict error
  • Output is continuous: "how hostile" rather than "hostile: true"
  • Deterministic from graph state — replay is trivially correct

weaknesses

  • Cosine similarity is slower than a flat component read — hot paths may still need a thin ECS layer
  • Dimension count requires empirical tuning; no principled derivation
  • Embedding updates on graph write are not free

opportunities

  • "Find entities in a similar relational position to X" becomes nearest-neighbor — no traversal
  • Embedding drift toward hostility is a signal players could surface via skills or abilities
  • Same infrastructure serves world graph, meta-graph, and cross-graph boundary evaluation

threats

  • Debugging "why does this NPC hate me" requires inspecting an embedding space — opaque without tooling
  • GM tools for causal explanation must be built proactively, not retrofitted
  • Embedding drift under high-frequency writes may accumulate error
04 Change Propagation → Influence Diffusion

Replaces the subscription system, generation-based cascade queues, and zombie listener management with continuous influence diffusion evaluated on demand. Systems query current graph state when they need it; the diffusion equation is the propagation model. No observer registration, no firing order ambiguity.

strengths

  • No subscription lifecycle — no registration, no deregistration, no leaks
  • No cascade ordering ambiguity; diffusion is commutative
  • Stateless — same inputs always produce same outputs
  • Political graph evolution IS diffusion — one model, multiple purposes

weaknesses

  • Full diffusion evaluation costs O(graph size) — cannot run on every tick
  • Some propagation genuinely requires immediacy that diffusion latency cannot satisfy
  • Convergence time on large or dense graphs needs eigenvalue analysis to predict

opportunities

  • Stubbornness parameter per faction models ideological rigidity as a continuous variable
  • Convergence rate (second-largest eigenvalue of trust matrix) is a tunable world texture knob
  • Slow convergence means factions hold divergent views for extended periods — which is interesting

threats

  • Immediacy requirements (combat events, direct player actions) may force a thin subscription layer anyway
  • Diffusion model unfamiliar to most game engineers — onboarding cost
  • Hard to explain to a player why their reputation "hasn't updated yet" without clear UX signaling
05 Temporal Model → Seeded Deterministic Replay

Replaces temporal indexing, snapshot storage, and save-state reconstruction with an append-only event log plus a seed. Any time T is reproducible by replaying the log from the seed. The cross-graph time delay mechanic is a replay offset — one mechanism serving propagation, information asymmetry, and persistence simultaneously.

strengths

  • Save files are event logs only — no snapshot overhead
  • Any time T is reproducible; time-travel debugging is free
  • The cross-graph delay mechanic is log-offset arithmetic, not a separate system
  • Server crash recovery is log replay from last checkpoint

weaknesses

  • Replay to late T is expensive for long-running servers with large logs
  • Floating-point nondeterminism in Node.js can break replay determinism
  • Log compaction strategy required to prevent unbounded growth

opportunities

  • World seeds become meaningful — shareable, diffable, historically replayable
  • GM "rewind and investigate" is a log seek, not a feature build
  • Information asymmetry between graphs is log offset — political entities can be "behind" by design

threats

  • Any external random source or async ordering variation breaks replay — requires strict write discipline
  • Long logs mean cold start replay is slow; checkpointing strategy must be designed carefully
  • Players exploiting seed knowledge for metagaming if seeds are ever exposed
06 Agent Epistemic State → Dempster-Shafer Belief Functions

Replaces NPC knowledge flags, binary known/unknown fields, and fog-of-war as a separate system with formal belief functions. Each agent holds a distribution over graph state: committed belief, committed disbelief, and explicit ignorance (mass on "I don't know"). Different agents holding contradictory beliefs about the same fact is correct by design, not a bug to be resolved.

strengths

  • Structured ignorance is a formal value — "I don't know" is not a missing field
  • Agent beliefs combine rigorously via Dempster's rule of combination
  • Outdated beliefs and incorrect beliefs are both expressible without special casing
  • The investigation mechanic has a mathematical foundation: surface portions of the belief function

weaknesses

  • Dempster's rule produces counterintuitive results under highly conflicting evidence
  • Computationally heavier than boolean flags — not suitable for per-tick evaluation
  • Requires a new mental model for content authors and QA

opportunities

  • Investigation as a gameplay verb — players find out what NPCs believe, not what is true
  • Propagation delay between graphs creates agents with legitimately outdated but internally consistent beliefs
  • Belief divergence between factions is a conflict driver that emerges from the model, not from scripting

threats

  • Hard to QA "does this NPC believe the right thing" without belief function inspection tooling
  • Players may perceive NPC ignorance as AI stupidity rather than intentional design
  • Dempster's rule can amplify conflicting evidence into unexpected results under adversarial input
07 Cross-Graph Federation → DAG-Bounded Boundary Verbs

Replaces shared state, implicit API contracts, and eventual consistency patches with a typed boundary verb whitelist forming a directed acyclic graph at the inter-graph level. Cross-graph references are explicit writes using permitted verb types only. Consistency between graphs is a sheaf condition — local data must agree at overlap or the inconsistency is deliberate and named.

strengths

  • Inter-graph coupling is always explicit — no silent cross-bleed
  • Circular inter-graph dependencies are rejected at boot by topological sort
  • Each graph instance has independent characteristics (temporal indexing, persistence strategy)
  • The boundary is the interface specification — self-documenting

weaknesses

  • Boundary verb taxonomy must be actively maintained as domains evolve
  • Sheaf consistency conditions may be over-specified for the actual use cases
  • Cross-graph ID resolution requires a registry — another component to keep correct

opportunities

  • Time delay between graphs is a torn consistency condition — formally definable, exploitable as player-facing mechanics
  • Intelligence, spies, and information asymmetry are structural properties of the boundary, not scripted events
  • Each graph can be swapped, mocked, or versioned independently — strong testability

threats

  • Boundary verb proliferation — "just one more" cross-graph edge until the boundary dissolves
  • Requires active governance discipline; no technical enforcement of boundary cleanliness over time
  • Sheaf formalism may be unnecessary complexity if graphs are few and stable
08 Cognitive Capacity & Memory → Hyperdimensional Computing

Replaces NPC memory as an explicit list, forget-after-N-events rules, and manual memory management with high-dimensional vector bundles. Each NPC's relational context is a superposition of bound entity-verb-entity triples. Superposition is inherently lossy — older and weaker associations become harder to retrieve as new ones are added. Forgetting emerges from the math, not from a designed decay function.

strengths

  • Forgetting is a mathematical property, not a lifecycle to manage
  • Capacity limits are emergent — no explicit memory budget per NPC
  • Retrieval fidelity degrades gracefully; NPCs misremember rather than fail
  • Interference between memories produces confusion and uncertainty naturally

weaknesses

  • Specific memory retention cannot be guaranteed — plot-critical facts may be lost
  • Retrieval is probabilistic — hard to test deterministically
  • Unusual implementation surface; no off-the-shelf Node.js HDC library

opportunities

  • Embedding dimension becomes a character attribute — high-memory NPCs are meaningfully different
  • Interference can model trauma, confusion, or cognitive load as emergent states
  • NPCs who have witnessed many events literally cannot recall early ones — this is correct and interesting

threats

  • Debugging memory failures is opaque — requires specialized inspection tooling
  • Content authors needing predictable NPC recall will fight the probabilistic model
  • Players may perceive memory interference as bugs, not as emergent cognition
